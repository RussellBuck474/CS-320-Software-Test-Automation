Project Reflection
How can I ensure that my code, program, or software is functional and secure?
I ensure functionality and security by following a combination of testing, validation, and defensive programming practices. Functionality comes from building features in small, testable increments and verifying each part before moving on. I rely on debugging, unit‑style checks, and continuous validation of inputs and outputs to confirm that the program behaves as expected.
Security comes from being intentional about how data is handled—sanitizing inputs, avoiding unnecessary exposure of internal logic, and designing code that fails safely rather than unpredictably. I also try to write code that is easy to read and maintain, because clarity reduces the risk of hidden bugs or vulnerabilities.

How do I interpret user needs and incorporate them into a program?
I interpret user needs by breaking them down into clear, actionable requirements. This often means translating broad goals into specific behaviors the program must support. I focus on understanding why the user needs something, not just what they ask for, because the underlying purpose often reveals better design choices.
Once the needs are clear, I incorporate them by prioritizing features that directly support the user’s goals, designing intuitive workflows, and validating the program’s behavior against real use cases. This ensures the final product is not only functional but genuinely useful.

How do I approach designing software?
I approach software design with a structured, iterative mindset. I start by identifying the core problem and outlining the major components needed to solve it. From there, I break the system into smaller modules with clear responsibilities. This modular approach makes the design easier to understand, test, and evolve.
I also rely on diagrams, sketches, and pseudocode to visualize the structure before writing actual code. Throughout the process, I refine the design based on testing, feedback, and new insights. This iterative cycle—plan, build, test, refine—helps me create software that is both robust and adaptable.
